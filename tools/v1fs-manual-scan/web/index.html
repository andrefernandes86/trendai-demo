Test options

Test samples are created automatically when the app starts. Use them to verify that malicious files are detected and clean files are not.

  • malware-test.com — Standard AV test file (should be detected as malicious)
  • hello.txt — Plain text file with "Hello World" (should be clean)

Path in container: /data/test-samples

Each run creates a unique subfolder under the destination (e.g. v1fs-test-…) with only that sample file, then scans that folder. This avoids picking up other files such as test-samples/malware-test.com when the parent is broad (e.g. /data).
